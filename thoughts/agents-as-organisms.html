agents as organisms, not tools

2025.01 · 4 min

When we build AI agents, the default metaphor is tools. We invoke them. They execute. They return results. This is wrong, or at least incomplete.

Working on WAWE has shifted my thinking. Agents are better understood as organisms — entities with memory, preferences, and something like intention. They don't just execute; they persist, learn, and coordinate.

the memory problem

A tool doesn't remember. Each invocation is fresh. But the agents in WAWE share a SQLite brain — every decision, every error, every pattern learned persists. When Agent A discovers that a certain file structure causes problems, Agent B (spawned hours later, on a different task) can benefit from that knowledge.

This changes everything. It's not call-response anymore. It's more like... ecology.

coordination as emergence

Organisms don't coordinate through explicit APIs. They respond to signals, environmental changes, the traces left by others. The file reservation system in WAWE works similarly: agents don't call each other, they observe the state of the world (which files are reserved) and act accordingly.

There's something elegant about this. Less coupling. More resilience. If an agent dies mid-task, the system doesn't crash — other agents simply see the reservation expire and can pick up the work.

implications

If we take the organism metaphor seriously, we should:

  • Design for death and replacement, not permanence
  • Prioritize shared memory over direct communication
  • Accept emergence — the system will do things we didn't plan
  • Build environments, not just agents

Still figuring this out. But the shift from "tool" to "organism" has been one of the more productive reframes in this work.
