coordination without control

2024.11 · 5 min

Most systems assume coordination requires control. A central server. A master process. A coordinator node. But some of the most robust systems coordinate without any central authority.

Git is peer-to-peer. Bitcoin has no master node. Ant colonies have no CEO. These systems achieve coordination through protocol, not hierarchy.

stigmergy revisited

Ants coordinate by modifying their environment. One ant leaves a pheromone trail; another follows it; the trail gets stronger. No ant tells another what to do. The environment is the communication channel.

In software terms: the database is the message bus. You don't call other services — you write to shared state and let others react.

WAWE and environmental coordination

This is the model behind WAWE's agent coordination. Agents don't call each other. They read from and write to a shared SQLite database. File reservations, task queues, learnings — all stored in the environment.

When Agent A reserves a file, Agent B simply sees the reservation (by querying the database) and acts accordingly. No direct communication needed. If Agent A crashes, the reservation eventually expires. The system heals itself.

the tradeoffs

This approach isn't always better. You lose:

  • Immediate feedback (you poll instead of push)
  • Strong consistency (eventual consistency is the norm)
  • Debugging clarity (no call stack, just state changes)

But you gain:

  • Resilience (no single point of failure)
  • Flexibility (add new agents without changing existing ones)
  • Simplicity (each agent is independent)

Not every system should work this way. But when building systems that need to be robust and evolvable, it's worth considering: can we coordinate through environment rather than control?
