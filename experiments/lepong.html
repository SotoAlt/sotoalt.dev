What is JEPA? Joint Embedding Predictive Architecture — a neural network that learns to predict the next state of a scene from pixel observations. Instead of generating future frames (like a video model), it predicts in a compressed latent space, making it fast enough to run in real time.

This experiment: the left paddle is controlled entirely by a 13M-parameter JEPA. Every ~6 times per second, this page captures a 128x128 screenshot of the game, sends it to a server running the neural network, and gets back a predicted ball position. The paddle moves to that prediction. No physics engine, no game state — the model only sees pixels and must figure out where the ball is going.

01

pixels in, paddle out

Every ~6 Hz the client renders a 128×128 frame, optionally occludes the right side, and sends the PNG to the server. The server runs a frozen 13M-param CNN JEPA (encoder → predictor → state head) and returns the predicted ball_y as the paddle target. 1,930 trainable params. No game state, only pixels.

02

the occlusion toggle

At 0% the task is trivial — any tracker that sees the ball wins. Toggle to 40% (press e) and the right side goes dark before the model sees it. The model must imagine the ball through the blind zone. You still see the full court via the purple overlay. Watch the cyan JEPA ball and the error metrics as occlusion increases.

03

what the metrics show

Live p50 / p95 / max error for JEPA vs a classical physics extrapolator, for both ball_y and ball_x. The classical baseline has ground truth (unfair advantage) — it's an oracle for comparison, not a competitor. Press R to reset the error buffers.

about

The left paddle is controlled by a 13M-parameter CNN JEPA that sees only pixels — no game state, no physics engine. It takes 3 frames of history, predicts where the ball will be next, and moves the paddle there. Trained on 30K Pong frames with a frozen encoder and a 1930-parameter state head.

Code and research journal at github.com/SotoAlt/aura.
